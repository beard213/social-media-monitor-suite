# 系统边界与数据保护

本系统面向公开内容的事件级监测、广告过滤、内容线索扩展和人工复核。

## 支持的处理

- 按关键词、平台、短视频/直播、时间和区域发现公开内容；
- 对公开标题、描述、媒体转写、评论和直播消息进行内容分析；
- 从评论主题、话题标签和关联公开内容生成匿名化内容线索；
- 保存公开来源、采集时间、处理记录和文件哈希；
- 经人工复核后推送事件结果和媒体证据。

## 不实现的处理

- 不根据政治观点、申诉经历、学生身份等给具体个人建立“高危”档案；
- 不把“上访”“不满”“思想偏差”等表述直接作为人员风险结论；
- 不通过关注/朋友关系推断现实身份、政治倾向或社会关系；
- 不做跨平台现实身份合并与持续追踪；
- 不以模型判断替代人工事实核验或人员处置决定。

## 数据最小化

默认写入前匿名化作者标识，不保存原始昵称、头像、主页、IP、精确位置或联系方式。需要保留原始证据时，应由有权限的独立证据库依据适用法律、访问控制、保存期限和审计要求管理。
