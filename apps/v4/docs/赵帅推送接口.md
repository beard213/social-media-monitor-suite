# 结果推送接口

在 `.env` 配置：

```env
PUSH_ENABLED=true
PUSH_EVENTS_URL=http://目标服务/api/v1/monitor/events
PUSH_MEDIA_URL=http://目标服务/api/v1/monitor/media
PUSH_BEARER_TOKEN=token
```

## 1. 事件结果

核心系统向 `PUSH_EVENTS_URL` 发送JSON：

```json
{
  "event_id": "content_123",
  "platform": "douyin",
  "content_type": "video",
  "title": "公开内容标题",
  "description": "公开描述",
  "source_url": "https://...",
  "published_at": "2026-07-14T10:00:00+08:00",
  "filter_status": "kept",
  "risk_status": "suspicious",
  "matched_keywords": ["拖欠工资"],
  "region_tags": ["雄安新区"],
  "topic_template": "labor_rights",
  "audits": [
    {
      "modality": "metadata_comments",
      "status": "suspicious",
      "labels": ["labor_dispute"],
      "risk_words": [],
      "confidence": 0.86
    }
  ],
  "evidence": [
    {"type": "video", "sha256": "..."}
  ]
}
```

目标端建议返回：

```json
{"success": true, "ack_id": "ack_123"}
```

系统会保存推送状态、请求摘要哈希、尝试次数、错误信息和 `ack_id`。

## 2. 媒体文件

核心系统向 `PUSH_MEDIA_URL` 发送 `multipart/form-data`，包含内容ID和媒体文件。只有已经完成授权媒体采集并生成证据记录时才会上传。

## 3. “推送”与“实时推流”的区别

当前工程实现的是：

- 事件结果API推送；
- 已采集媒体文件上传。

如果赵帅要求RTMP/SRT实时转推，还需另行提供：

- RTMP或SRT目标地址；
- stream key或鉴权信息；
- 视频编码、分辨率、码率要求；
- 断线重连和并发限制；
- 是否转推完整直播或仅推送命中风险的时间段。
