# 界面与任务流程

## 1. 新建监测任务

进入“监测任务”后按三步配置：

1. **监测范围**：任务名称、主题模板、平台、短视频/直播、区域、发布时间和排序；
2. **搜索条件**：关注词、排除词、匹配方式、结果数量和执行频率；
3. **处理策略**：评论采集、内容线索扩展、文字检测、媒体采集和推送策略。

“立即执行”只触发关键词发现。评论、扩展、检测、媒体采集和推送由任务策略自动加入队列。

## 2. 内容发现

内容发现页支持以下下拉筛选：

- 监测任务；
- 平台；
- 短视频/直播；
- 区域（包含雄安新区）；
- 主题；
- 内容过滤状态；
- 处理阶段。

每条内容显示阶段：`已发现 → 已抓评论 → 已扩展 → 已检测 → 已采集 → 已推送`。

## 3. 线索扩展

线索扩展只生成内容级线索：

- 评论中重复出现的话题；
- 话题标签；
- 连接器返回的关联公开内容。

线索可标记为已复核、忽略或提升为新的监测关键词，不保存或展示真实个人身份。

## 4. 直播监测

关键词任务发现直播后，可启动连续120秒分片。真实切片要求连接器返回有效 `stream_url`，服务器已安装FFmpeg，且Worker持续运行。

## 5. 推送

“推送给赵帅”当前指事件JSON和媒体文件接口推送。若实际要求RTMP/SRT实时转推，还需单独配置目标地址、鉴权、转码参数和断线重连策略。
