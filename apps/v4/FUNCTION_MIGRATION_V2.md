# 新治理前端与旧系统功能迁移说明（V2）

## 一、整合原则

本版本以领导认可的“AI安全治理平台”视觉和页面结构为主，不再继续沿用旧系统的页面外观。旧系统后端、数据库和任务链路尽量原样保留，通过API迁入新前端。

页面对应关系：

| 新前端页面 | 已迁入的旧系统能力 |
|---|---|
| 态势大屏 | 总览统计、内容数量、直播会话、证据数量、检测数量、任务趋势、热点话题、最新内容、服务状态 |
| 监测配置 | 抖音/快手连接器状态、连接测试、关键词任务创建/运行/暂停/删除、直播间ID或链接接入 |
| 直播监控 | 直播会话列表、视频证据播放、音频/文本证据关联、ASR转写查看、检测结果、评论/弹幕、后台运行日志、停止会话 |
| 预警处置 | 内容列表与详情、人工确认、忽略、评论采集、公开关系采集、内容扩展、重新检测、结果推送、报告导出 |
| 预警处置-账号线索 | 原风险账号列表、账号公开内容/互动统计、公开行为时间线、关系线索 |
| 预警处置-话题线索 | 原话题排行、词云、关键词聚合、关联内容和线索依据 |
| 系统管理 | 健康检查、连接器契约、后台任务队列、推送记录、API Key配置、能力边界说明 |

## 二、已经直接迁入并可使用的功能

1. **总览和统计**
   - `/api/console/overview`
   - `/api/health`
   - `/api/contents`
   - `/api/live-sessions`
   - `/api/jobs`

2. **关键词监测任务**
   - 创建任务
   - 任务模板和区域选择
   - 启用/暂停
   - 手动运行
   - 删除任务
   - 评论采集、线索扩展、自动检测、自动采集和推送配置

3. **直播ID接入**
   - 支持抖音、快手和演示平台
   - 支持直接粘贴 `https://live.douyin.com/<房间ID>`
   - 自动提取直播间ID
   - 支持配置60、120、300秒分片
   - 复用旧抖音直播采集桥接流程

4. **直播结果查看**
   - 直播会话状态
   - 视频片段播放
   - ASR文本文件查看
   - 音频、元数据和证据文件关联
   - 检测模型、版本、标签、置信度和真实跳过状态
   - 评论和授权直播消息
   - 相关后台任务和错误信息

5. **内容处置和扩展**
   - 待确认、已确认、已忽略
   - 采集评论
   - 获取公开关系
   - 扩展关联内容
   - 重新文本检测
   - 推送给下游系统
   - 导出HTML分析报告

6. **话题和账号线索**
   - 热点话题和词云
   - 风险账号公开内容与互动汇总
   - 账号公开行为时间线
   - 内容、话题、评论账号和公开关系依据

7. **运行管理**
   - 搜索、评论、关系、扩展、检测、直播分片和推送任务队列
   - 成功、等待、运行中和失败状态
   - 尝试次数、载荷和错误原因
   - 连接器接口契约与环境配置状态

## 三、可以复用，但本次仍属于“适配实现”的功能

| 功能 | 当前实现 | 后续建议 |
|---|---|---|
| 预警事件 | 由内容风险、检测结果和人工状态聚合显示 | 新增正式 `AlertEvent` 表和状态流转 |
| 直播分片 | 通过同一内容下的视频、音频、文本、元数据证据文件关联 | 新增 `LiveSegment`，统一segment_id、开始/结束时间和处理状态 |
| ASR转写 | 读取TXT证据文件 | 增加句级时间戳和视频时间轴联动 |
| 实时刷新 | 页面定时轮询API | 增加WebSocket或SSE推送 |
| 账号配置 | 显示后端环境变量和连接器状态 | 新增平台账号、授权有效期、权限范围和Token刷新管理 |
| 报告 | 浏览器即时生成HTML | 新增报告表、版本、审核人、PDF/Word和服务器存档 |
| 操作日志 | 当前主要展示后台Job和推送记录 | 增加用户级操作审计表 |

## 四、仍需新增开发的能力

### 1. 平台账号与授权管理

当前抖音、快手配置依赖 `.env`：

- `DOUYIN_CONNECTOR_URL`
- `DOUYIN_CONNECTOR_TOKEN`
- `KUAISHOU_CONNECTOR_URL`
- `KUAISHOU_CONNECTOR_TOKEN`

需要新增：

- 平台账号表
- 账号别名和平台UID
- 授权范围
- 授权有效期
- Token刷新
- 敏感字段加密
- 前端新增/编辑/停用接口

### 2. 真实直播弹幕

当前旧抖音项目主要完成视频、音频和ASR，没有真实弹幕源。需要平台授权连接器将公开直播消息推送到：

```text
POST /api/provider/ingest/live-message
```

### 3. 正式直播分片模型

建议新增：

```text
segment_id
session_id
content_id
start_time
end_time
duration
video_evidence_id
audio_evidence_id
text_evidence_id
metadata_evidence_id
audit_status
risk_status
```

### 4. 正式预警和处置闭环

建议新增：

```text
AlertEvent
AlertReview
AlertAction
Report
```

支持：待确认、审核中、已确认、已忽略、已上报、已归档。

### 5. 用户、角色和审计

当前登录页只承担前端入口和API Key保存，不是真正身份认证。正式系统需要：

- 用户登录
- Token/Session
- 管理员、审核员、查看员角色
- 页面和操作权限
- 完整操作日志

## 五、部署结构

```text
frontend/                  React源码，可继续二次开发
web/index.html             已构建生产入口
web/static/assets/         已构建JS/CSS，无需服务器安装Node即可运行
web/legacy-v1/             上一版旧前端备份
app/                       旧系统后端、任务、数据库模型和服务全部保留
```

## 六、结论

旧系统多数业务能力可以直接迁入新前端，核心后端无需推倒重做。本版本已完成第一轮实际迁移。后续开发重点不是重复做已有任务和采集功能，而是补齐平台账号持久化、真实弹幕、分片实体、正式预警、报告存档和用户权限。
